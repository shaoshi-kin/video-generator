## 第二部分：生成素材（图文 → 视频）

### 步骤 5：用 Midjourney 生成图片

**工具**: [Midjourney Discord](https://discord.com/channels/@me) 或 [Leonardo.ai](https://leonardo.ai)

**操作流程**:

```
1. 打开 video_project_kimi.json
2. 找到 "image_prompts" 部分
3. 复制第一个提示词到 Midjourney
4. 生成图片后下载保存
5. 按分镜顺序命名：
   - scene_01_hook.png  (开头钩子)
   - scene_02_story.png (故事展开)
   - scene_03_data.png  (数据展示)
   - ...以此类推
```

**提示词示例**（从脚本中获取）:
```
Cinematic financial news style, PricewaterhouseCoopers logo shattered, 
10 billion dollars in red numbers floating, dark background with 
spotlight effects, professional business atmosphere, 8k quality
```

---

### 步骤 6：用 Runway/Luma 图生视频

**工具选择**:
- [Runway Gen-3](https://runwayml.com)（推荐，效果最佳）
- [Luma Dream Machine](https://lumalabs.ai/dream-machine)（免费额度多）
- [Pika Labs](https://pika.art)（简单易用）

**操作流程**:

```
1. 打开 video_project_kimi.json
2. 找到 "video_prompts" 部分
3. 选择对应的图片上传
4. 粘贴视频提示词
5. 设置参数:
   - Motion Bucket: 40-60 (中等运动)
   - Duration: 4秒 (每个镜头)
6. 生成视频并下载
7. 命名格式: clip_01.mp4, clip_02.mp4...
```

**关键镜头建议**:
- 开头（0-3秒）：必须做视频，吸引眼球
- 数据展示：可做动态数字动画
- 结尾：简单转场即可

## 第三部分：剪辑合成

### 步骤 7：剪映/ CapCut 剪辑

**推荐工具**: [剪映专业版](https://www.capcut.cn)（免费，功能全）

**操作流程图**:

```
┌─────────────────────────────────────────────────────────┐
│  1. 新建项目 → 选择 9:16 竖屏比例                        │
│                      ↓                                   │
│  2. 导入素材 → 按顺序拖入时间轴                           │
│     - 视频片段 (clip_01.mp4, clip_02.mp4...)            │
│     - 图片 (scene_01.png, scene_02.png...)              │
│                      ↓                                   │
│  3. 添加配音 → 导入 "full_script" 文案                   │
│     - 使用剪映"文本朗读"功能                             │
│     - 或自己录音                                        │
│                      ↓                                   │
│  4. 添加字幕 → 识别字幕或手动输入                         │
│     - 根据 script.scenes 中的 subtitle 字段             │
│     - 调整字体、颜色、位置                               │
│                      ↓                                   │
│  5. 添加 BGM → 根据 "bmg_suggestion" 选择               │
│     - 剪映音乐库搜索关键词                               │
│     - 音量调至 20-30% 不盖过人声                         │
│                      ↓                                   │
│  6. 导出视频 → 1080P, 30fps, MP4格式                     │
└─────────────────────────────────────────────────────────┘
```

**字幕排版建议**（根据脚本）:
```
每行不超过 10 个字
字体：粗体黑体
颜色：白色 + 黑色描边
位置：屏幕下方 1/3 处
动画：逐字出现或淡入
```

## 常见问题 FAQ

**Q1: 脚本运行很慢怎么办？**
- Kimi API 有时限流，多等一会儿
- 文章太长时自动使用 128k 模型，会更慢但保证不截断

**Q2: 生成的图片不满意？**
- 在 Midjourney 中修改提示词
- 添加风格词：`cinematic`, `8k`, `professional lighting`
- 使用 `--s 750` 参数增加风格化

**Q3: 视频生成有瑕疵？**
- Runway 中调整 Motion Bucket（30-60之间测试）
- 复杂动作生成不稳定时，改用静态图片+剪映动画

**Q4: 一天能做多少条？**
- 纯手动：1-2 条/天
- 半自动（本脚本）：3-5 条/天
- 全自动（未来）：10+ 条/天（需接入批量API）
