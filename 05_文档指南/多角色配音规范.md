## 一、快速开始

在文章中用 `@音色:` 标记每句话的朗读者：

```markdown
# AI与未来工作

@全局:女声
@默认图: 背景

@新闻男: 大家好，欢迎收看今天的科技前沿节目。

@女声: 今天我们来讨论人工智能。

@男声: AI正在改变我们的工作方式。

回到女声总结。
（空行后自动回到全局默认：女声）
```

## 二、核心规则

### 2.1 基本格式

```
@音色名: 要说的话
```

**重要规则（v2.0）**：
- **空行分隔段落** - 每段以空行结束
- **空行重置** - 空行后音色和图片都回到 `@全局` 设置的默认值
- `@全局:音色` 设置整篇文章的默认音色

### 2.2 标记位置

**方式A：标记和内容在同一行（推荐）**
```markdown
@新闻男: 这是第一段内容。

@女声: 这是第二段内容。
```

**方式B：标记和内容分开（跨行）**
```markdown
@新闻男:
这是第一段内容。

@女声:
这是第二段内容。
```

**方式C：音色持久化（不换行）**
```markdown
@新闻男: 这是第一段。
@新闻男: 这是第二段，同一音色继续。
```

## 三、全局设置与重置规则

### 3.1 全局设置

```markdown
@全局:女声
@默认图: 吃饭
```

**作用**：
- `@全局:女声` → 空行后自动回到女声
- `@默认图: 吃饭` → 空行后自动回到吃饭图片

### 3.2 空行重置规则

```markdown
@全局:女声
@默认图: 吃饭

@新闻男: 第一段，新闻男 + 吃饭。

第二段，女声 + 吃饭。
（空行后 → 自动回到全局默认：女声 + 吃饭）

@云夏: @图:睡觉 第三段，云夏 + 睡觉。

第四段，女声 + 吃饭。
（空行后 → 自动回到全局默认：女声 + 吃饭）
```

### 3.3 持续化设置（不换行）

如果不想重置，不要加空行：

```markdown
@新闻男: 这是第一段。
这是第一段继续，还是新闻男。
@图:桌子 还是第一段，新闻男 + 桌子。

现在空行了，回到女声 + 吃饭。
```

## 六、常见问题

### Q1: 不标记音色会怎样？
**A**: 使用 `@全局` 设置的默认音色。没设置全局则使用命令行默认（Xiaoxiao）。

### Q2: 标记了错误的音色名？
**A**: 会显示警告并使用默认音色：
```
⚠️  未知音色 '机器人'，使用默认音色
```

### Q3: 为什么空行后音色变了？
**A**: 这是 v2.0 的设计。空行后音色重置为 `@全局` 设置的默认值。如果不需要重置，把内容放在同一行，不要加空行。

### Q4: 视频模式支持多音色吗？
**A**: 支持，所有段落合并成一个音频文件，音色自然切换。

### Q5: 如何设置默认语速？
**A**: 命令行参数 `--rate` 对所有音色生效：
```bash
python3 video_generator_pro.py -p projects/XXX --rate +10%
```

### Q6: Windows 创建的 .md 文件能用吗？
**A**: 支持！自动处理 Windows CRLF 换行符（`\r\n`）。

### Q7: `@新闻男:` 和 `@新闻男: `（冒号后空格）有区别吗？
**A**: 没有区别，都支持。`@新闻男: 内容` 和 `@新闻男:内容` 效果相同。

## 七、最佳实践

### 7.1 双人对话建议
- 设置 `@全局` 为一方（如女声）
- 另一方用 `@音色:` 临时插入
- 空行后自动回到全局设置

### 7.2 多人内容建议
- 使用 `@全局` 设置主要讲述者
- 其他人用 `@音色:` 临时插入
- 每个发言后空行，自动回到主讲述者

### 7.3 内容长度建议
- 每个音色段落不要太短（至少10字）
- 频繁切换会让听众疲劳
- 建议每段保持3-5句话
