# 灵活图片分配规范

**版本**: 2.0
**更新日期**: 2025-04-26
**关键更新**: 空行后图片重置为全局默认值，支持中文文件名

---

## 一、功能概述

灵活图片分配功能允许你：
- **指定段落使用特定图片** - 如 `@图:吃饭`
- **设置默认图片** - 未指定时使用默认图
- **重复使用同一张图片** - 多个段落共用一张图
- **支持中文文件名** - `吃饭.jpg`、`睡觉.png` 等

---

## 二、核心规则

### 2.1 基本格式

| 标记 | 作用 | 示例 |
|------|------|------|
| `@图:名称` | 指定使用某张图片 | `@图:吃饭` 使用 吃饭.jpg |
| `@图片:名称` | 同上 | `@图片:睡觉` 使用 睡觉.jpg |
| `@img:名称` | 同上（英文） | `@img:bg` 使用 bg.jpg |
| `@默认图:名称` | 设置全局默认图片 | `@默认图:吃饭` |
| `@默认图片:名称` | 同上 | `@默认图片:背景` |

**重要规则（v2.0）**：
- **空行分隔段落** - 每段以空行结束
- **空行重置** - 空行后图片回到 `@默认图` 设置的值
- **冒号后空格** - 支持 `@图: 吃饭` 或 `@图:吃饭`

### 2.2 图片引用方式

**中文名称（推荐）**：
```markdown
@图:吃饭    → 使用 吃饭.jpg
@图:睡觉    → 使用 睡觉.png
@图:打豆豆  → 使用 打豆豆.jpg
```

**数字编号**：
```markdown
@图:01    → 使用 scene_01.jpg 或 01.jpg
@图:1     → 同上（自动补零）
```

**英文名称**：
```markdown
@图:bg.jpg    → 使用 bg.jpg
@图:封面      → 使用 封面.jpg
```

## 三、分配规则详解

### 3.1 空行重置规则

```markdown
@全局:女声
@默认图: 吃饭

@新闻男: 第一段，新闻男 + 吃饭（默认）。

@图:睡觉
@新闻男: 第二段，新闻男 + 睡觉（指定）。

第三段，女声 + 吃饭。
（空行后 → 图片回到默认：吃饭）

@图:打豆豆
@云夏: 第四段，云夏 + 打豆豆（指定）。

第五段，女声 + 吃饭。
（空行后 → 图片回到默认：吃饭）
```

### 3.2 标记位置

**方式A：标记和内容在同一行**
```markdown
@图:吃饭 这是第一段，使用吃饭图片。

@图:睡觉 这是第二段，使用睡觉图片。
```

**方式B：标记和内容分开（跨行）**
```markdown
@图:吃饭
这是第一段，使用吃饭图片。

@图:睡觉
这是第二段，使用睡觉图片。
```

**方式C：图片持久化（不换行）**
```markdown
@图:吃饭 这是第一段。
这是第一段继续，还是吃饭图片。

现在空行了，图片回到默认值。
```

### 3.3 与音色标记结合

```markdown
@全局:女声
@默认图: 吃饭

@新闻男: 第一段，新闻男 + 吃饭。

@图:睡觉
@新闻男: 第二段，新闻男 + 睡觉。

第三段，女声 + 吃饭（空行后重置）。

@图:打豆豆
@云夏: 第四段，云夏 + 打豆豆。

第五段，女声 + 吃饭（空行后重置）。
```

## 四、使用示例

### 示例1：中文文件名（推荐）

```markdown
# 美食介绍

@全局:女声
@默认图: 餐厅环境

@男声: 大家好，欢迎来到美食探店。

@图:招牌菜
@男声: 这是我们的招牌菜，色香味俱全。

回到女声介绍餐厅环境。

@图:厨师
@男声: 我们的大厨有20年经验。

@图:食材
@女声: 所有食材都是当天采购。

@图:价格
@男声: 价格非常亲民。

@女声: 欢迎来品尝！
```

**图片分配**：
- 第1段 → 餐厅环境.jpg（默认）
- 第2段 → 招牌菜.jpg（指定）
- 第3段 → 餐厅环境.jpg（空行后默认）
- 第4段 → 厨师.jpg（指定）
- 第5段 → 食材.jpg（指定）
- 第6段 → 价格.jpg（指定）
- 第7段 → 餐厅环境.jpg（空行后默认）

### 示例2：基础用法（顺序分配）

不指定图片，按文件名排序自动分配：

```markdown
# 新闻播报

今天第一条新闻：股市大涨。
上证指数突破3000点，创近期新高。

第二条新闻：国际油价回落。
布伦特原油价格跌至80美元每桶。

第三条新闻：央行降准。
释放流动性约5000亿元。
```

**图片分配**（假设 03_images/ 中有3张图）：
- 第1段 → 第1张图
- 第2段 → 第2张图
- 第3段 → 第3张图

### 示例3：重复使用图片

```markdown
# 产品介绍

@全局:女声
@默认图: 产品

@男声: @图:Logo 开场展示Logo。

产品介绍，使用默认图产品。

@图:Logo 结尾再次展示Logo。

@女声: 感谢观看！
```

### 示例4：复杂场景（多音色+多图片）

```markdown
# 科技访谈

@全局:女声
@默认图: 工作室

大家好，欢迎收看科技前沿。
（女声 + 工作室）

@男声: @图:嘉宾 我是今天的嘉宾王教授。

@女声: @图:数据 首先看看最新数据。

这是趋势图表。
（空行后 → 女声 + 工作室）

@男声: @图:嘉宾 我认为这个趋势会持续。

@女声: 感谢教授分享。
（空行后 → 女声 + 工作室）
```

## 八、故障排查

### Q1: 指定了图片但没有生效？
**A**: 检查图片是否存在于 `03_images/` 目录，文件名要完全匹配（不含扩展名）。

### Q2: 如何查看图片分配结果？
**A**: 运行时会显示分配信息：
```
🖼️ 场景 1: 使用指定图片 [吃饭] -> 吃饭.jpg
🖼️ 场景 2: 使用指定图片 [睡觉] -> 睡觉.png
```

### Q3: 可以跨目录引用图片吗？
**A**: 目前只搜索 `03_images/`, `02_manual_images/`, `01_api_images/`。

### Q4: 默认图和指定图的关系？
**A**: 
- `@默认图:xx` 设置全局默认值
- 段落开头无 `@图:` 时，空行后使用默认图
- 段落开头有 `@图:yy` 时临时覆盖

### Q5: 为什么空行后图片变了？
**A**: 这是 v2.0 的设计。空行后图片重置为 `@默认图` 设置的值。如果不需要重置，把内容放在同一行，不要加空行。

### Q6: 支持 Windows 创建的 .md 文件吗？
**A**: 支持！自动处理 Windows CRLF 换行符（`\r\n`）。

## 九、与多音色功能结合

两个功能可以同时使用：

```markdown
# 双人对话

@全局:女声
@默认图: 工作室

主持人开场白。
（女声 + 工作室）

@男声: @图:专家 专家发言。

@女声: @图:数据 展示数据。

@男声: 专家点评。
（空行后 → 男声 + 工作室）

@女声: 主持人总结。
（空行后 → 女声 + 工作室）
```

**效果**：
- 音色：女声 → 男声 → 女声 → 男声 → 女声
- 图片：工作室 → 专家 → 数据 → 工作室 → 工作室
