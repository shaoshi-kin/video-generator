## 二、两种工作模式详解

### 2.1 图片模式（Ken Burns动画）

**适用场景**：文章配图、PPT讲解、静态图片展示

**特点**：
- ✅ 文章每段落生成一段音频
- ✅ 图片自动匹配音频段落数
- ✅ Ken Burns 慢速缩放动画
- ✅ 最高质量编码（veryslow + crf 15）

**运行流程**：
```
01_article/文章.md → edge-tts生成 → 05_audio/scene_01~N.mp3
03_images/scene_01~N.jpg + 音频 → Ken Burns动画 → 合成视频
```

### 2.2 视频模式（原画画质保留）

**适用场景**：实拍素材配音、视频混剪、保留原始画质

**特点**：
- ✅ 整篇文章合并为一个音频
- ✅ 视频直接复制（`-c:v copy`）不损失画质
- ✅ 音频驱动时长（视频自动循环）
- ✅ AI配音替换原视频音频

**运行流程**：
```
01_article/文章.md → edge-tts生成 → 05_audio/scene_01.mp3（完整版）
04_videos/原视频 + 完整音频 → 循环播放 → 合成视频
```

## 三、文章标记规范（v4.0）

### 3.1 核心规则

| 标记 | 作用 | 格式 |
|------|------|------|
| `@全局:音色` | 设置全局默认音色 | `@全局:女声` |
| `@默认图:名称` | 设置全局默认图片 | `@默认图:吃饭` |
| `@图:名称` | 指定当前段落图片 | `@图:睡觉` |
| `@音色:内容` | 指定当前段落音色 | `@男声: 你好` |

**重要规则**：
- **空行分隔段落** - 每段以空行结束
- **空行重置** - 空行后音色和图片都回到全局默认值
- **标记位置** - `@图:` 和 `@音色:` 放在段落第一行
- **冒号后空格** - 支持 `@图: 吃饭` 或 `@图:吃饭`

### 3.2 使用示例

#### 示例1：基础用法
```markdown
# 文章标题

@全局:女声
@默认图: 吃饭

@新闻男: 这是第一段，使用默认图吃饭。

@图:睡觉
@新闻男: 这是第二段，强制使用睡觉图片。

这段继续用默认图片，默默声音。
（空行后 → 女声 + 吃饭）
```

**效果**：
- 第1段：新闻男 + 吃饭
- 第2段：新闻男 + 睡觉（指定）
- 第3段：女声 + 吃饭（空行后重置）

#### 示例2：复杂场景
```markdown
@全局:女声
@默认图: 背景

@男声: 开场白，男声 + 默认图背景。

@图:图表
@男声: 展示数据，男声 + 图表。

回到女声讲解，图片回到默认背景。
（空行后 → 女声 + 背景）

@图:产品
@女声: 介绍产品，女声 + 产品图。

@图:价格
@男声: 公布价格，男声 + 价格图。

结尾总结，回到女声和默认图。
```

### 4.1 图片引用方式

| 方式 | 示例 | 说明 |
|------|------|------|
| **数字编号** | `@图:01` | 找 `scene_01.jpg` 或 `01.jpg` |
| **中文名称** | `@图:吃饭` | 找 `吃饭.jpg` 或 `吃饭.png` |
| **英文名称** | `@图:bg` | 找 `bg.jpg` |

### 4.2 图片命名规范

**推荐命名（最直观）**：
```
03_images/
├── 吃饭.jpg       ← @图:吃饭
├── 睡觉.png       ← @图:睡觉
├── 打豆豆.jpg     ← @图:打豆豆
├── 桌子.jpg       ← @图:桌子
└── 背景.jpg       ← @默认图:背景
```

**也支持传统命名**：
```
03_images/
├── scene_01.jpg
├── scene_02.jpg
├── 01.jpg
└── 02.jpg
```

### 4.3 分配规则

```
@全局:女声          ← 全局默认音色

@图:睡觉           ← 指定图片
@新闻男: 第二段     ← 新闻男 + 睡觉（指定）
```

## 五、完整使用教程

### 5.1 创建新项目

#### 图片模式项目
```bash
# 1. 创建项目文件夹
mkdir -p projects/my_article/{01_article,03_images}

# 2. 写入文章
cat > projects/my_article/01_article/文章.md << 'EOF'
# AI正在改变世界

@全局:女声
@默认图: 背景

@男声: 大家好，我是主持人。

@图:数据
@男声: 看看最新数据。

回到女声总结。

@图:结尾
@女声: 感谢观看。
EOF

# 3. 放入图片（命名要对应）
cp 背景.jpg projects/my_article/03_images/
cp 数据.png projects/my_article/03_images/
cp 结尾.jpg projects/my_article/03_images/

# 4. 一键生成
python3 01_核心脚本/video_generator_pro.py -p projects/my_article --voice Xiaoxiao
```

#### 视频模式项目
```bash
# 1. 创建项目文件夹
mkdir -p projects/my_video/{01_article,04_videos}

# 2. 写入文章
cat > projects/my_video/01_article/文案.md << 'EOF'
# 摄影技巧

@女声: 今天分享摄影技巧。
EOF

# 3. 放入原视频
cp my_footage.mp4 projects/my_video/04_videos/scene_01.mp4

# 4. 一键生成
python3 01_核心脚本/video_generator_pro.py -p projects/my_video --voice Yunyang
```

### 5.2 命令行参数

```bash
python3 01_核心脚本/video_generator_pro.py \
    -p projects/my_project \           # 项目路径（必需）
    --voice Xiaoxiao \                 # AI音色（默认Xiaoxiao）
    --rate +0% \                       # 语速（默认+0%）
    --subtitle \                       # 添加字幕
    --subtitle-style news \            # 字幕风格
    --transition fade \                # 转场效果
    --resolution 1920x1080 \           # 分辨率
    --fps 30                           # 帧率
```

## 六、参数详解

### 6.1 图片模式编码参数（最高质量）

```python
-c:v libx264        # H.264编码
-preset veryslow    # 最慢预设 = 最佳质量
crf 15              # 接近无损画质
-tune stillimage    # 针对静态图片优化
-b:a 256k           # 高码率音频
-movflags +faststart # 网络播放优化
```

### 6.2 视频模式编码参数（画质保留）

```python
-c:v copy           # 直接复制视频流，零损失
-c:a aac -b:a 192k  # 音频重新编码
-map 0:v:0 -map 1:a:0  # 视频+新音频
-stream_loop N      # 视频循环（音频更长时）
```

## 七、常见问题

### Q1: 图片数量和音频段落不匹配？
**A**: 图片不够时会复用已有图片。也可以通过 `@图:` 指定复用某张图。

### Q2: 视频比音频长怎么办？
**A**: 自动以音频时长为准，视频会截断到音频结束。

### Q3: 音频比视频长怎么办？
**A**: 自动计算循环次数，使用 `-stream_loop` 让视频循环播放直到音频结束。

### Q4: 如何只生成音频不合成视频？
**A**: 运行后按 Ctrl+C 中断，或单独使用 article_to_voice_edge.py：
```bash
python3 01_核心脚本/article_to_voice_edge.py 文章.md --voice Xiaoxiao --output output.mp3
```

### Q5: 已有音频如何跳过生成？
**A**: 只要 `05_audio/` 目录存在且包含 `.mp3` 文件，就会自动跳过音频生成步骤。

### Q6: 为什么音色没按我写的来？
**A**: 检查是否有空行。空行会重置音色为 `@全局` 设置的默认值。如果不想重置，把内容放在同一行：
```markdown
@新闻男: 这是第一段。
@新闻男: 这是第二段。  # 不要空行，音色保持新闻男
```

### Q7: 支持 Windows 创建的 .md 文件吗？
**A**: 支持！自动处理 Windows CRLF 换行符（`\r\n`）。
