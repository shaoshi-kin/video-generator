## 二、两种工作模式详解

### 2.1 图片模式（Ken Burns动画 + 转场特效）

**适用场景**: 文章配图、PPT讲解、静态图片展示

**特点**:
- 文章每段落生成一段音频（并行3并发，v6.0）
- 图片自动匹配音频段落数
- Ken Burns 慢速缩放动画
- 场景淡入淡出（可配置时长）
- 段落间转场特效（28种可选）
- 转场音效自动混入（可选）
- 最高质量编码（veryslow + crf 15）
- 增量更新：只重建变更场景（v6.0）
- 素材预检：生成前检查完整性（v6.0）

**运行流程**:
```
01_article/文章.md → edge-tts并行生成 → 05_audio/scene_01~N.mp3
03_images/图片 + 音频 + 字幕 + 水印 + 淡入淡出 → 场景片段 → 06_scenes/
06_scenes/ + 转场特效 + BGM → 07_final/最终视频
```

### 2.2 视频模式（原画画质保留）

**适用场景**: 实拍素材配音、视频混剪、保留原始画质

**特点**:
- 整篇文章合并为一个音频
- 视频直接复制（`-c:v copy`）不损失画质
- 音频驱动时长（视频自动循环）
- AI配音替换原视频音频
- 字幕/水印/BGM 同样支持

**运行流程**:
```
01_article/文章.md → edge-tts生成 → 05_audio/scene_01.mp3（完整版）
04_videos/原视频 + 完整音频 + 字幕 + 水印 → 合成视频
```

## 三、文章标记规范（v6.0）

### 3.1 核心规则

| 标记 | 作用 | 格式 | 示例 |
|------|------|------|------|
| `@全局:音色` | 设置全局默认音色 | `@全局:女声` | `@全局:Yunyang` |
| `@默认图:名称` | 设置全局默认图片 | `@默认图:吃饭` | `@默认图: 背景` |
| `@图:名称` | 指定当前段落图片 | `@图:睡觉` | `@图: 数据` |
| `@音色:内容` | 指定当前段落音色 | `@男声:你好` | `@新闻男: 大家好` |

**重要规则**:
- **空行分隔段落** - 每段以空行结束
- **空行重置** - 空行后音色和图片都回到全局默认值
- **标记位置** - `@图:` 和 `@音色:` 放在段落第一行
- **冒号后空格** - 支持 `@图: 吃饭` 或 `@图:吃饭`
- **中文文件名** - 图片名直接用中文，如 `吃饭.jpg`

### 3.2 使用示例

#### 示例1：基础用法
```markdown
# 文章标题

@全局:女声
@默认图: 吃饭

@新闻男: 这是第一段，使用默认图吃饭。

@图:睡觉
@新闻男: 这是第二段，强制使用睡觉图片。

这段继续用默认图片，女声。
（空行后 → 女声 + 吃饭）
```

**效果**:
- 第1段：新闻男 + 吃饭
- 第2段：新闻男 + 睡觉（指定）
- 第3段：女声 + 吃饭（空行后重置）

#### 示例2：复杂场景
```markdown
@全局:女声
@默认图: 背景

@男声: 开场白，男声 + 默认图背景。

@图:图表
@男声: 展示数据，男声 + 图表。

回到女声讲解，图片回到默认背景。
（空行后 → 女声 + 背景）

@图:产品
@女声: 介绍产品，女声 + 产品图。

@图:价格
@男声: 公布价格，男声 + 价格图。

结尾总结，回到女声和默认图。
```

## 五、完整使用教程

### 5.1 创建新项目（推荐 --init）

#### 方式一：命令行初始化（自动生成配置和示例）

```bash
# 图片模式项目（带示例文章和图片目录）
python3 01_核心脚本/video_generator_pro.py --init --template news

# 视频模式项目
python3 01_核心脚本/video_generator_pro.py --init --template video
```

执行后会：
1. 创建项目文件夹结构
2. 生成 `.video_config.json`（带默认配置）
3. 生成示例 `文章.md`
4. 生成 `参数说明.md`
5. 生成默认 `watermark.png`

#### 方式二：手动创建

```bash
# 图片模式项目
mkdir -p projects/my_article/{01_article,03_images}

# 视频模式项目
mkdir -p projects/my_video/{01_article,04_videos}
```

### 5.2 命令行参数（v6.0 完整版）

```bash
python3 01_核心脚本/video_generator_pro.py \
    -p projects/my_project \              # 项目路径（必需）
    --voice Xiaoxiao \                    # AI音色
    --rate +18% \                         # 语速
    --subtitle \                          # 添加字幕
    --subtitle-style youtube \            # 字幕风格
    --subtitle-animation slide_up \       # 字幕动画
    --subtitle-mode sentence \            # 字幕模式: sentence(逐句) / full(整段)
    --scene-fade 0.3 \                    # 场景淡入淡出时长
    --transition pixelize \               # 转场效果
    --transition-duration 0.8 \           # 转场时长
    --resolution 1920x1080 \              # 分辨率
    --fps 30 \                            # 帧率
    --watermark watermark.png \           # 水印路径
    --watermark-position bottom-right \   # 水印位置
    --bgm-volume 0.25 \                   # BGM音量
    --sfx \                               # 开启转场音效
    --normalize-audio \                   # 响度标准化到 -14 LUFS
    --intro-text "欢迎观看" \              # 片头文字（v6.0）
    --outro-text "感谢观看" \              # 片尾文字（v6.0）
    --dual-version \                      # 同时生成横竖双版本（v6.0）
    --generate-copy \                     # 生成发布文案（调用Kimi API，v6.0）
    --skip-pre-check \                    # 跳过素材预检（v6.0）
    --import-ppt ppt文件.pptx \            # 导入PPT为项目（v6.0）
    --no-parallel \                       # 关闭并行生成（v6.0）
    --batch \                             # 批量处理多个项目
```

### 5.3 配置优先（推荐）

**无需长命令行！** 所有参数写进 `.video_config.json`：

```json
{
  "voice": "Xiaoxiao",
  "resolution": "1920x1080",
  "fps": 30,
  "subtitle": true,
  "subtitle_style": "youtube",
  "subtitle_animation": "slide_up",
  "subtitle_mode": "sentence",
  "scene_fade": 0.3,
  "transition": "pixelize",
  "transition_duration": 0.8,
  "rate": "+18%",
  "bgm_volume": 0.25,
  "watermark": "watermark.png",
  "watermark_position": "bottom-right",
  "sfx": false,
  "dual_version": false
}
```

改完配置，直接运行：
```bash
python3 01_核心脚本/video_generator_pro.py -p projects/我的项目
```

临时覆盖（优先级高于配置）：
```bash
python3 01_核心脚本/video_generator_pro.py -p projects/我的项目 \
    --transition fade \
    --scene-fade 0.5
```

## 六、特效与功能详解

### 6.1 字幕系统（v6.0 重大升级）

#### 字幕显示模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `sentence` | **逐句显示**（默认）：按 `。！？；` 拆分，读完一句立即消失，下一句接上 | 口播、讲解 |
| `full` | **整段显示**：段落全部文字一次性显示 | 诗歌、短句 |

切换方式：
```json
{"subtitle_mode": "sentence"}  // 默认
{"subtitle_mode": "full"}      // 整段
```

#### 字幕风格
| 风格 | 说明 | 外观 |
|------|------|------|
| `news` | 新闻风格（底部居中，白字黑边） | 稳重正式 |
| `youtube` | YouTube风格（底部偏上，大字醒目） | 默认推荐 |
| `tiktok` | TikTok风格（底部大字，描边粗） | 短视频风 |
| `minimal` | 极简风格（小字，不抢眼） | 轻量简洁 |

#### 字幕动画
| 动画 | 说明 |
|------|------|
| `none` | 无动画，直接出现 |
| `slide_up` | 从下方滑入（推荐） |
| `fade_in` | 淡入效果 |

#### 字幕位置
- `news` 风格：底部居中，y_offset = 100
- `youtube` 风格：底部偏上，y_offset = 80
- `tiktok` 风格：底部居中，y_offset = 120
- `minimal` 风格：底部居中，y_offset = 100

### 6.2 转场特效（28种）

支持的所有转场效果：

| 效果 | 中文翻译 |
|------|----------|
| `fade` | 淡入淡出 |

| `wiperight` | 向右擦除 |
| `wipeup` | 向上擦除 |

| `slideleft` | 向左滑动 |
| `slideright` | 向右滑动 |
| `slideup` | 向上滑动 |
| `slidedown` | 向下滑动 |
| `pixelize` | 像素化 |
| `diagtl` | 对角线（左上到右下） |
| `diagtr` | 对角线（右上到左下） |
| `diagbl` | 对角线（左下到右上） |
| `diagbr` | 对角线（右下到左上） |
| `hlslice` | 水平切片 |
| `hrslice` | 水平切片（反向） |

### 6.3 场景淡入淡出

每个场景片段开头和结尾自动添加淡入淡出效果：
- `scene_fade: 0.3` → 开头淡入0.3秒，结尾淡出0.3秒
- `scene_fade: 0` → 关闭淡入淡出
- 推荐值：0.2 ~ 0.5

### 6.4 水印/Logo

- 默认启用，替换 `watermark.png` 即可更换
- 支持位置：`top-left`, `top-right`, `bottom-left`, `bottom-right`, `center`
- 默认透明度：0.75
- 水印路径支持相对路径（基于项目目录）或绝对路径

### 6.5 转场音效（SFX）

- 开启后自动读取 `02_sfx/` 目录下的音效文件
- 在转场时自动混入
- 多个音效文件会随机选择
- 适合添加 whoosh、切换等音效增强节奏感

### 6.6 片头片尾（v6.0）

自动生成纯色背景 + 文字的视频片段：
```bash
--intro-text "欢迎观看"   # 在第一个场景前插入
--outro-text "感谢关注"   # 在最后一个场景后插入
```

特点：
- 自动匹配当前分辨率
- 默认时长 3 秒
- 带淡入淡出效果
- 确保与主场景视频参数一致（fps、分辨率）

### 6.7 音频响度标准化（v6.0）

```bash
--normalize-audio
```

- 将所有音频统一标准化到 **-14 LUFS**（YouTube 推荐标准）
- 使用 ffmpeg `loudnorm` 滤镜
- 适合批量项目保持音量一致

### 6.8 横竖双版本输出（v6.0）

```bash
--dual-version
```

- 当前项目为横屏（1920x1080）时，额外生成竖屏版本（1080x1920）
- 当前项目为竖屏（1080x1920）时，额外生成横屏版本（1920x1080）
- 竖屏版本自动适配字幕位置和大小

### 6.9 编码参数

#### 图片模式（最高质量）
```python
-c:v libx264        # H.264编码
-preset veryslow    # 最慢预设 = 最佳质量
crf 15              # 接近无损画质
-tune stillimage    # 针对静态图片优化
-b:a 256k           # 高码率音频
-movflags +faststart # 网络播放优化
```

#### 视频模式（画质保留）
```python
-c:v copy           # 直接复制视频流，零损失
-c:a aac -b:a 192k  # 音频重新编码
-map 0:v:0 -map 1:a:0  # 视频+新音频
-stream_loop N      # 视频循环（音频更长时）
```

- 项目列表管理（创建/加载/刷新）
- 文章在线编辑（带标记语法提示）
- 图片批量上传（支持多选）

**兼容**: Gradio 3.x / 4.x / 5.x / 6.x

### 7.2 增量更新（断点续传升级版）

**原理**:
- 生成时记录每个场景的输入文件 MD5 和渲染参数到 `.build_manifest.json`
- 下次生成时对比：输入文件和参数均未变更 → 跳过重建
- 大幅提速：只重建变更的场景

**触发重建的情况**:
- 文章段落数变化（新增/删除段落）
- 图片文件内容变化（MD5 不同）
- 音频文件内容变化
- 配置参数变化（分辨率、字幕样式、转场等）
- 强制重建：`--regenerate-audio` 或删除 `06_scenes/.build_manifest.json`

- 图片/视频素材是否充足
- 字体是否可用
- 水印文件是否存在
- BGM 目录是否有音乐
- 输出目录是否可写

跳过检查：`--skip-pre-check`


### 7.6 插件系统

在项目目录下创建 `plugins/` 文件夹，编写 `.py` 文件：

```python
def post_parse_article(segments):
    """文章解析后钩子"""
    for seg in segments:
        seg['text'] = seg['text'].replace('旧词', '新词')
    return segments

def pre_generate_audio(segment):
    """单段音频生成前钩子"""
    print(f"准备生成: {segment['text'][:20]}...")
    return segment
```

支持钩子：
- `post_parse_article` - 文章解析后
- `pre_generate_audio` - 音频生成前
- `post_generate_audio` - 音频生成后
- `pre_build_scene` - 场景构建前
- `post_build_scene` - 场景构建后
- `pre_merge` - 合并转场前
- `post_merge` - 合并转场后

### 7.7 并行生成

**音频并行**（图片模式）：
- 使用 `asyncio.Semaphore(3)` 限制并发
- 最多同时生成 3 段音频
- 网络失败自动重试 3 次（等待 2s/4s/6s）

**场景并行**（图片模式）：
- 使用 `ProcessPoolExecutor(max_workers=3)`
- 最多同时渲染 3 个场景
- 完成后按场景 index 排序再合并，保证顺序正确

关闭并行：`--no-parallel`

### 7.8 运行日志

每次生成自动保存详细日志到 `07_final/generate_YYYYMMDD_HHMMSS.log`：
- 命令行参数
- 素材分配详情
- 各阶段耗时统计
- 场景详情（时长、大小）
- 失败原因（如有）

## 八、常见问题

### Q1: 为什么空行后音色变了？
**A**: 空行会重置音色为 `@全局` 设置的默认值。如果不想重置，把内容放在同一行，不要加空行。

### Q2: 支持中文图片名吗？
**A**: 支持！`吃饭.jpg` 可以用 `@图:吃饭` 引用。

### Q3: Windows 创建的 .md 文件能用吗？
**A**: 支持！自动处理 Windows CRLF 换行符。

### Q4: 修改了水印/字幕配置但没生效？
**A**: `06_scenes/` 下的旧场景文件不带新效果，删除它们重新生成：
```bash
rm projects/我的视频/06_scenes/*.mp4
```

### Q5: 如何关闭字幕？
**A**: 改 `.video_config.json` 里的 `"subtitle": false`，或命令行：
```bash
python3 01_核心脚本/video_generator_pro.py -p projects/我的视频 --no-subtitle
```

### Q6: 逐句字幕和整段字幕怎么切换？
**A**: 改 `.video_config.json`：
```json
{"subtitle_mode": "sentence"}  // 逐句显示（默认）
{"subtitle_mode": "full"}      // 整段显示
```

### Q7: 图片数量和音频段落不匹配？
**A**: 图片不够时会复用已有图片。也可以通过 `@图:` 指定复用某张图。

### Q8: 视频比音频长怎么办？
**A**: 自动以音频时长为准，视频会截断到音频结束。

### Q9: 音频比视频长怎么办？
**A**: 自动计算循环次数，使用 `-stream_loop` 让视频循环播放直到音频结束。

### Q10: 如何只生成音频不合成视频？
**A**: 运行后按 Ctrl+C 中断，或单独使用 article_to_voice_edge.py：
```bash
python3 01_核心脚本/article_to_voice_edge.py 文章.md --voice Xiaoxiao --output output.mp3
```

### Q11: 已有音频如何跳过生成？
**A**: 只要 `05_audio/` 目录存在且包含 `.mp3` 文件，就会自动跳过音频生成。但如果文章段落数变了，会自动清理旧音频重新生成。

### Q12: 为什么音色没按我写的来？
**A**: 检查是否有空行。空行会重置音色为 `@全局` 设置的默认值。如果不想重置，把内容放在同一行：
```markdown
@新闻男: 这是第一段。
@新闻男: 这是第二段。  # 不要空行，音色保持新闻男
```

### Q13: Web UI 启动报错 localhost 503？
**A**: 已在代码中自动设置 `no_proxy`，如仍报错检查系统代理设置：
```bash
export no_proxy=localhost,127.0.0.1
python3 01_核心脚本/webui.py
```

### Q14: 增量更新不生效，每次都全量重建？
**A**: 检查是否修改了配置参数（如分辨率、字幕样式等），这些变化会触发重建。如需强制全量重建：
```bash
rm projects/我的视频/06_scenes/.build_manifest.json
rm projects/我的视频/06_scenes/*.mp4
```
